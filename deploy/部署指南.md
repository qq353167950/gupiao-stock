# 📖 手把手部署指南

> 本文档覆盖两种部署方式 + Cloudflare 自定义域名配置，每一步都写明**打开哪里、点什么、填什么**。
>
> - **方式 A**：Docker 一键部署（自己有 VPS 时最推荐，5 条命令搞定）
> - **方式 B**：panel.adkynet.com 面板部署（Pterodactyl 容器）
> - **第三部分**：Cloudflare Tunnel 自定义域名（两种方式通用）
> - **第四部分**：部署后验证清单

---

# 方式 A：Docker 一键部署（推荐）

**适用**：任何装有 Docker 的 Linux 服务器（阿里云/腾讯云/境外 VPS 均可）。

## A1. 安装 Docker（已装则跳过）

SSH 登录服务器，执行：

```bash
# 一键安装 Docker（官方脚本，大陆服务器自动用国内源）
curl -fsSL https://get.docker.com | bash
# 启动并设置开机自启
sudo systemctl enable --now docker
# 验证（能输出版本号即成功）
docker compose version
```

## A2. 上传项目

```bash
# 方式一：git（项目在你自己的仓库时）
cd /opt && git clone <你的仓库地址> stock-analyzer-web

# 方式二：scp 上传（本地打包后传）
# 本地执行: scp stock-analyzer-web.zip root@服务器IP:/opt/
# 服务器执行: cd /opt && unzip stock-analyzer-web.zip
```

## A3. 填配置

```bash
cd /opt/stock-analyzer-web
cp .env.example .env
nano .env
```

**最少只改这 3 行**（其余全部保持默认即可）：

```ini
PUBLIC_BASE_URL=https://stock.你的域名.com     # 没有域名先填 http://服务器IP:8888
NOTIFY_CHANNELS=serverchan                     # 推送渠道
SERVERCHAN_SENDKEY=SCT开头的一串               # 第三部分 C0 教你申请
```

有自定义域名的话再加一行（token 获取见第三部分）：

```ini
CF_TUNNEL_TOKEN=eyJhIjoi开头的长字符串
```

按 `Ctrl+O` 回车保存，`Ctrl+X` 退出。

## A4. 一键启动

```bash
docker compose up -d --build
```

首次构建约 5-10 分钟（下载镜像 + 安装依赖）。完成后：

```bash
# 看启动日志（按 Ctrl+C 退出查看，不影响服务）
docker compose logs -f
```

看到下面这些行就是成功了：

```
✅ 定时任务调度器已启动
   - 收盘后分析：每交易日 15:10
   - 早盘推送：  每交易日 08:20
🚀 启动 Web 服务: http://0.0.0.0:8888
```

## A5. 日常运维命令

```bash
docker compose logs -f              # 看日志
docker compose restart              # 重启
docker compose down                 # 停止
docker compose up -d --build        # 更新代码后重建
docker compose exec stock-analyzer python -m app.skill_manager check   # 查 skill 版本
docker compose exec stock-analyzer python -c "from app import notifier; notifier.send('测试','ok')"  # 测推送
```

> 数据库、日志、报告、skill 都挂载在项目目录下（`./data` `./logs` `./skills`），
> 容器删了重建也不丢数据。

# 方式 B：panel.adkynet.com 面板部署

> AdKyNet 面板基于 Pterodactyl。界面语言可能是法语/英语，下文括号标注两种写法。

## B1. 创建 Python 服务器实例

1. 打开 `https://panel.adkynet.com/` 并登录
2. 如果你还没有实例：去 AdKyNet 官网购买 **VPS / 应用托管** 类型产品时，
   应用类型选择 **Python**（不要选游戏服/Discord bot 专用类型）
3. 登录面板后，首页服务器列表里**点击你的服务器名称**进入管理页

## B2. 上传项目文件

1. 本地准备 zip 包：把整个项目打包，**排除**这些目录（体积大且无用）：
   `venv/`、`data/*.db`、`skills/.repo/`、`__pycache__/`、`.git/`
2. 面板左侧菜单点 **Files**（Fichiers / 文件）
3. 点右上角 **Upload**（Téléverser / 上传）→ 选择你的 zip
4. 上传完成后，在文件列表里**右键点击 zip 文件** → 选 **Unarchive**（Décompresser / 解压）
5. 确认解压后根目录（`/home/container`）下能直接看到 `start.py`、`app/`、`requirements.txt`
   > 如果解压出来多了一层文件夹（如 `stock-analyzer-web/start.py`），
   > 全选里面的文件 → 右键 Move → 移动到 `/home/container` 根目录

## B3. 配置启动参数

1. 左侧菜单点 **Startup**（Démarrage / 启动）
2. 找到 **App py file**（或 `PY_FILE` / `MAIN_FILE` 之类的变量）→ 填：
   ```
   start.py
   ```
3. 如果有 **Additional Python packages** / **Requirements file** 变量 → 填：
   ```
   requirements.txt
   ```
4. 如果 egg 没有依赖安装变量，找到页面顶部的 **Startup Command** 输入框，整个替换为：
   ```bash
   if [ ! -f .deps-installed ]; then python -m pip install --no-cache-dir -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple || python -m pip install --no-cache-dir -r requirements.txt; touch .deps-installed; fi; python start.py
   ```
   > 作用：首次启动自动装依赖（3-8 分钟），以后凭 `.deps-installed` 标记跳过

## B4. 配置环境变量

面板的 Startup 页签下方通常有 Variables 区域，但自定义变量数量有限。
**最省事的方式：直接传 `.env` 文件**（效果完全相同）：

1. 本地把 `.env.example` 复制一份改名 `.env`，填好这几行：
   ```ini
   PUBLIC_BASE_URL=https://stock.你的域名.com
   NOTIFY_CHANNELS=serverchan
   SERVERCHAN_SENDKEY=你的SendKey
   CF_TUNNEL_TOKEN=eyJhIjoi...        # 见第三部分
   MX_APIKEY=                          # 法国服务器强烈建议配（见 B7）
   ```
2. 面板 **Files** → **Upload** 把 `.env` 传到根目录

## B5. 查看分配端口（配 Tunnel 要用）

1. 左侧菜单点 **Network**（Réseau / 网络）
2. 记下 **Primary**（主端口）那一行的端口号，比如 `25565`
   > 这就是容器的 `SERVER_PORT`，第三部分 CF Tunnel 的 Service URL 要填它

## B6. 启动

1. 回到服务器 **Console**（控制台）页
2. 点右上角 **Start**（Démarrer / 启动）按钮
3. 首次启动等依赖安装（Console 会滚动 pip 输出），最终看到：
   ```
   🚀 启动 Web 服务: http://0.0.0.0:25565
   ✅ 定时任务调度器已启动
   ```

## B7. ⚠️ 法国服务器数据源问题（重要）

AdKyNet 服务器在欧洲，A 股数据源（东财/雪球/腾讯）对境外 IP 经常限流。
**部署后先手动测一只股票**（Console 里执行）：

```bash
python -c "import asyncio; from app.task_manager import create_task; import asyncio
async def t():
    tid = create_task('600519', depth='quick')
    print('任务已建:', tid)
    await asyncio.sleep(600)
asyncio.run(t())"
```

如果日志里大量 fetcher 超时/失败 → 在 `.env` 加 `MX_APIKEY=你的东财妙想Key` 后重启。
仍不行 → 建议改用方式 A 部署在大陆 VPS。

# 第三部分：Cloudflare 自定义域名（Tunnel）

> **为什么用 Tunnel 而不是普通 DNS 解析**：面板给的是"IP+随机端口"，CF 代理不支持
> 非标端口；Tunnel 由容器主动连出去，不需要开放任何端口，自动 HTTPS，IP 变了也不用改。
> Docker 部署同样推荐 Tunnel（免去配 nginx + 证书）。

## C0. 前置：申请 Server酱 SendKey（推送用，1 分钟）

1. 打开 `https://sct.ftqq.com` → 微信扫码登录
2. 点顶部 **SendKey** 菜单 → 复制 `SCT` 开头的字符串
3. 这就是 `.env` 里 `SERVERCHAN_SENDKEY` 的值

## C1. 域名接入 Cloudflare（已接入则跳过）

1. 打开 `https://dash.cloudflare.com` → 注册/登录（免费版即可）
2. 首页点 **添加站点**（Add a site）→ 输入你的裸域名（如 `example.com`）→ 继续
3. 选 **Free 免费计划** → 继续
4. CF 会显示两个 **NS 服务器地址**（形如 `xxx.ns.cloudflare.com`）
5. 去你的**域名注册商**（阿里云/腾讯云/GoDaddy...）后台 → 域名管理 → DNS/NS 设置
   → 把 NS 改成 CF 给的那两个 → 保存
6. 回到 CF 点 **检查名称服务器**，等待生效（几分钟到几小时），
   收到"站点已激活"邮件即完成

## C2. 创建 Tunnel

1. 打开 `https://one.dash.cloudflare.com`（Cloudflare Zero Trust 控制台）
   > 首次进入会让你选一个团队名，随便起；套餐选 **Free**
2. 左侧菜单点 **Networks**（网络）→ **Tunnels**
3. 点 **Create a tunnel**（创建隧道）
4. 连接器类型选 **Cloudflared** → 点 **Next**
5. 隧道名称填 `stock-analyzer` → 点 **Save tunnel**
6. 进入 "Install and run a connector" 页面：
   - **不需要**按页面说的去装任何东西（项目已内置）
   - 找到页面中的安装命令，形如：
     `cloudflared service install eyJhIjoiXXXXXX...`
   - **只复制 `eyJhIjoi` 开头的那串长字符串**（这就是 token）
7. 把 token 填进 `.env`：
   ```ini
   CF_TUNNEL_TOKEN=eyJhIjoiXXXXXX...
   ```
8. 点页面底部 **Next** 进入下一步（先不要关页面）

## C3. 绑定域名（Public Hostname）

接上一步，在 "Route tunnel" 页面：

1. **Subdomain**（子域名）填：`stock`（想用别的也行）
2. **Domain**（域名）下拉选择你在 C1 接入的域名
3. **Path** 留空
4. **Service** 部分：
   - **Type** 选 `HTTP`
   - **URL** 填 `localhost:端口`：
     - Docker 部署 → `localhost:8888`
     - 面板部署 → `localhost:B5记下的端口`（如 `localhost:25565`）
5. 点 **Save tunnel**（保存）

> DNS 记录会自动创建，无需手动去 DNS 页面加记录。

## C4. 重启服务生效

- **Docker**：`docker compose restart`
- **面板**：Console 页点 **Restart**

日志里看到：

```
🌐 启动 Cloudflare Tunnel...
   ✓ Tunnel 已启动
```

回到 CF Zero Trust 的 Tunnels 列表，你的隧道状态变为绿色 **HEALTHY**。

## C5. 验证

浏览器访问 `https://stock.你的域名.com` → 能打开平台首页即成功（自动 HTTPS 🔒）。

最后把 `.env` 里 `PUBLIC_BASE_URL` 改成这个域名并重启——推送消息里的报告链接就会用它。

# 第四部分：部署后验证清单

按顺序执行，全部通过即部署完成：

| # | 验证项 | 命令/操作 | 预期 |
|---|---|---|---|
| 1 | 服务健康 | 浏览器开 `https://stock.你的域名.com/health` | `{"status":"ok",...}` |
| 2 | 推送渠道 | `python -c "from app import notifier; notifier.send('测试','部署成功')"` | 微信收到消息 |
| 3 | skill 版本 | `python -m app.skill_manager check` | 显示 commit 与"已是最新" |
| 4 | 单元测试 | `python tests/test_core.py` | `24 通过, 0 失败` |
| 5 | 单股分析 | 网页首页输入 `600519` 选"快速分析"提交 | ~10 分钟后出报告 |
| 6 | 定时链路 | 等一个交易日，08:20 收推送 | 微信收到当日推荐 |

> Docker 部署时命令前加 `docker compose exec stock-analyzer `，
> 面板部署直接在 Console 执行。

## 常见问题速查

| 症状 | 原因 | 解决 |
|---|---|---|
| Tunnel 状态 DOWN | token 填错/没重启 | 核对 `CF_TUNNEL_TOKEN` 完整复制，重启服务 |
| 域名 502 | Service URL 端口不对 | C3 第 4 步的端口改成实际监听端口 |
| 分析全部超时 | 境外 IP 被数据源限流 | `.env` 配 `MX_APIKEY`，或换大陆服务器 |
| 推送没收到 | 渠道/凭据没配对 | 跑验证项 2 看报错；检查 `NOTIFY_CHANNELS` 拼写 |
| skill 更新失败 | 服务器连不上 GitHub | `.env` 配 `GITHUB_PROXY=https://gh-proxy.com/` |
| 定时任务时间不对 | 时区不是北京时间 | 已修复：调度显式绑定北京时间，与服务器/面板时区无关 |
| 内存不足被杀 | 并发太高 | `.env` 调 `MAX_CONCURRENT_TASKS=1` |
